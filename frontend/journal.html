02

Design Decisions

DuckDB over PostgreSQL

Chose DuckDB as the local analytical engine for its columnar storage, zero-dependency deployment, and native Parquet support. The entire dataset fits in memory, making a client-server database unnecessary overhead.

HRP over Mean-Variance

Markowitz mean-variance optimization is notoriously unstable with estimated inputs. HRP avoids matrix inversion entirely, producing more robust allocations that degrade gracefully under estimation error.

Topological Backtest

The backtest engine processes the strategy DAG topologically, ensuring each node's dependencies are resolved before execution. This prevents look-ahead bias by construction rather than convention.

Cornish-Fisher over Gaussian VaR

Financial returns exhibit fat tails and negative skewness. The Cornish-Fisher expansion adjusts quantile estimates for non-normality, producing more realistic risk measures than the Gaussian assumption.

Static Archive Philosophy

The project is designed to sunset gracefully. All live telemetry degrades to static JSON, all charts are self-contained HTML files, and the frontend requires no backend to serve.

Brutalist Aesthetic

The visual language deliberately rejects the rounded-corner, pastel SaaS aesthetic. Sharp edges, high contrast, monospaced data, and film grain texture signal institutional seriousness over consumer friendliness.

03

Lessons Learned

01

Covariance estimation is the bottleneck

The quality of portfolio construction is bounded by the quality of the covariance estimate. GraphicalLasso with cross-validation was the single most impactful improvement to allocation stability.

02

Regime detection requires humility

HMMs provide probabilistic regime estimates, not certainties. The system works best when regime signals modulate weights gradually rather than triggering binary switches.

03

Backtest overfitting is the default outcome

Without PBO and DSR, every backtest looks good. These metrics should be computed first, not last. They determine whether the rest of the analysis is meaningful.

04

DAG-based execution prevents subtle bugs

Processing nodes in topological order eliminates an entire class of look-ahead and dependency-ordering bugs that plague sequential backtests.

05

Design for sunset from day one

Building the system to degrade gracefully to static files meant every component had a clear data contract. This constraint improved the architecture more than any optimization.
